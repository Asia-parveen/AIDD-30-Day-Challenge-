# 2Ô∏è)- Vibe Coding vs Specification-Driven Development

### Q1: Why does Vibe Coding usually create problems after one week?

### Answer:-

Vibe coding is liye problem ban jata hai kyunke isme start mein plan hi nahi hota. Banda
bas code likhta jata hai jahan dil kare. Pehle 2‚Äì3 din sab theek lagta hai, lekin 1 haftay
baad code itna messy ho jata hai ke samajh hi nahi aata ke kis jagah kya likha tha.

confuse feel hota hai. Yani planning na hone ki wajah se system tootna shuru ho jata
hai.

### Q2: How would Specification-Driven Development prevent those problems?

### Answer:-

Specification-Driven Development (SDD) ye issues is liye rok deta hai kyunke isme
coding se pehle clear plan ban jata hai. Yani pehle likh diya jata hai ke function kya
karega, system ka flow kya hoga, rules kya hain. Jab coding plan ke mutabiq hoti hai to
code clean, simple aur samajhne layak banta hai. Future mein changes bhi easy hoti
hain, aur bugs bhi kam aate hain. Yani SDD project ko organised aur stable bana deta
hai.
